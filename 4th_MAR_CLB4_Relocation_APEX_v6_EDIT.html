How to Use

Everything is editable. Units, buildings, labels, dates, phases, statuses, notes, KPI cards, phase cards, footer text - all of it. You can also add new units and buildings or delete existing ones as the plan changes. Nothing requires a developer.

Four ways to edit:

  1. Detail Drawer (recommended). Click any tile on the map, any building on the map, or any row in the Unit Roster / Building Schedule tables. The drawer slides in from the right. Click Edit to swap the read-only view for an editable form with every field (name, echelon, start/end building, phase, move date, status, notes, occupants, etc.). Click Save to write back to the data and propagate to the map, tables and charts. Click Delete to remove the record and its map graphic.
  2. Data Editor tab (bulk). Full CRUD tables for Units and Buildings. Add Row, delete row, edit any cell inline. Save Units / Save Buildings commits and syncs to the map.
  3. Edit Mode on the map. Press E or click Edit Mode. Every text element (headers, KPI labels, badges, card details, card dates) shows a dashed outline. Click any outlined element to edit it inline. Click a tile or building to open its visual edit panel (position, size, color, FY tag). Drag tiles to reposition them.
  4. Inline cells. The Editor tab cells accept text input directly without Edit Mode.

Tabs: Overview (1), Interactive Map (2), Phase Grid (3), Unit Roster (4), Building Schedule (5), Projects & Cost (6), Timeline (7), Charts (8), Data Editor (9). COA Comparison and Help are available from the sidebar and command palette.

COA toggle (Course of Action): Top-right of the toolbar - segmented switch COA 1 / COA 2. COA 1 (Primary) = current scope, the baseline plan. COA 2 (Alternate) = the proposed scope amendment from the PMO recommendation slide. Switching the toggle re-renders the Roster, Building Schedule, Timeline, Projects & Cost, Overview total, and any open detail drawer to show that COA's data. Selection persists across sessions. The Projects & Cost tab always shows BOTH totals on the hero card so you can see the cost delta at a glance.

COA Comparison tab: Side-by-side Primary vs Alternate cards for everything that differs - cost delta summary, Project scope additions / removals, Unit re-routes (e.g. CLB-4 Supply 3237 -> 3213 in COA 2), and Building role changes (3410/3413 swing-space designation, 3237 removed from scope). UNRESOLVED items (e.g. 4th MAR Regt Supply during the SCH-3235 DPRI 2-yr window) are flagged in red so unresolved gaps are visible at a glance.

Editing COA 2 fields: Open any unit / building / project in the detail drawer, click Edit. The form shows BOTH a COA 1 (Primary) section and a COA 2 (Alternate) section. Leave any alt field blank to inherit from Primary; fill it to override. The "Mark as unresolved" checkbox flags units (like 4th MAR Regt Supply) where the alt plan has a known gap.

Projects & Cost: Ten-row editable table of the 4th Marines Camp Schwab consolidation projects. Every cell (PAX ID, FIWEB ID, Building, Cost, Scope, Supported Unit, Completion, Delivery, Note) is editable free-text. The Note column takes any string; the conventions * (HN-funded FRF DEMO) and ** (HN-funded renovation) tint the row for quick scanning but are not enforced - write anything you want there. The Total at the top, in the footer, and on the Overview card recalculates live as you type any Cost value. Add Project appends a blank row; each row has a Delete button. Export CSV or JSON from the tab header.

Map demolition choreography: When a building is vacated (Phase 2 for Bldg 3446, Phase 4 for Bldg 3509) the building first takes a VACATED state (amber dashed border, line-through, "VACATED" label) for five seconds, then transitions to DEMOLISHED (red diagonal stripe pattern, "DEMOLISHED" label). Reset returns the buildings to active.

Timeline (Building Lifecycle): One row per building, five columns (one per phase). Each cell shows that building's state (Active / Final / Swing / Vacated / Demolished / Pending) plus the unit chips occupying it during that phase. Click any cell or row label to open the building detail drawer.

Interactive Map: Open the Map tab, press Space or → to advance through relocation phases, R to reset. The phase grid at the top highlights the active phase and pulses the matching card when a tile lands. FY tags pop up on moving tiles when they arrive. The Consolidated Armory tile does not exist before Phase 3 - it materializes at Bldg 1024 when consolidation happens. Bldg 3446 slides into the FRF demo zone when Phase 2 demolishes it.

Search / Filter: Toolbar search + Type / Phase / Status selects filter across Roster, Buildings, Timeline, and Editor tables. Reset clears all filters.

Command Palette: Ctrl+K / Cmd+K. Type to filter, ↑/↓ to navigate, Enter to run.

Theme: Press T to toggle dark/light. Saved between sessions.

Persistence: All edits auto-persist to browser localStorage under 4thMAR_edits_v1 and 4thMAR_data_v1. Data Editor → Clear All Overrides wipes both and reloads defaults.

Export / Import: Per-table CSV and JSON from the Roster and Buildings tabs. Export All JSON from the Data Editor. Export HTML Snapshot produces a complete standalone file with current edits baked in - hand it to anyone, they can open it in a browser, no server needed.

Print: P or Print button. Sidebar, toolbars, edit chrome, overlays and the map simulation are hidden. Only the active tab's content is printed on 17x11 landscape.
